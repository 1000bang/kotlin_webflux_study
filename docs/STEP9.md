# 9단계: 왜 코틀린을 쓰는가

## 목표
- **JVM 호환**, **간결한 문법**, **null 안전**, **Spring/Android 지원**, **실무 채택 이유**를 말로 정리할 수 있다.
- 채용·기술 선택·레거시 전환 시 "왜 코틀린인가"를 논리적으로 설명할 수 있다.

---


## 1. JVM 호환

- 코틀린은 **JVM 위에서** 돌아가고, **자바 바이트코드**로 컴파일된다.
- **기존 자바 라이브러리·Spring·메이븐/그레이들**을 그대로 쓸 수 있다. 자바 클래스와 코틀린 클래스가 같은 프로젝트에 섞여 있어도 된다.
- **점진적 전환**이 가능하다. 한 번에 자바를 버릴 필요 없이, 새 모듈·새 API만 코틀린으로 쓰고, 레거시는 그대로 두면서 좁혀 갈 수 있다.

---

## 2. 간결한 문법

- **보일러플레이트 감소**: data class, 타입 추론, 단일 표현식 함수, 람다로 같은 동작을 더 짧게 쓴다.
- **가독성**: "이 타입이 뭘 하는지"가 코드에 잘 드러나게 쓰기 좋다 (확장 함수, 스코프 함수, when 등).
- **유지보수**: 코드 양이 줄고 의도가 분명하면 리뷰·리팩터링이 수월해진다.

---

## 3. null 안전

- **타입에 nullable 여부**가 들어가서, 컴파일 단계에서 null 오용을 줄일 수 있다.
- **?. ?: let require** 등으로 "null일 때만", "null이면 예외"를 한 줄에 표현할 수 있어, 런타임 NPE를 줄이는 데 도움이 된다.
- 금융·결제·대규모 서비스처럼 **장애 비용이 큰** 곳일수록, "왜 코틀린을 쓰는가"에서 null 안전을 강조하는 경우가 많다.

---

## 4. Spring / Android 지원

- **Spring**: Spring Boot·Spring MVC/WebFlux가 코틀린을 공식 지원한다. 코틀린 DSL, null-safe API 활용이 가능하다.
- **Android**: 구글이 안드로이드 공식 언어로 코틀린을 채택했다. 백엔드와 같은 언어를 쓰면 **풀스택·모바일 연동** 시 이점이 있다.
- **공식 지원**이 있어서 문서·커뮤니티·버전 호환을 기대하기 쉽다.

---

## 5. 실무 채택 이유 (요약)

- **토스·카카오·당근·라인·우아한형제들** 등이 백엔드·안드로이드에 코틀린을 쓰는 사례가 많다.
- 채용공고에 **Java + Kotlin**이 함께 올라오고, **"코틀린 경험 우대"**가 붙는 경우가 늘었다.
- 실무에서 기대하는 것: **버그 감소**(null·타입), **개발 속도**(간결함·도메인 표현), **기존 투자 유지**(JVM·자바 호환).

---

## 6. 실전에서 어떻게 쓰는지

### 6-1. "왜 코틀린 써요?"라고 물었을 때

- **한 문장**: "JVM 호환이라 기존 자바랑 같이 쓸 수 있고, null 안전하고 문법이 짧아서 버그 줄이고 유지보수하기 좋아서요."
- **세 문장**으로 나누면: (1) JVM·자바 호환으로 점진적 전환 가능, (2) null 안전·간결한 문법으로 런타임 오류 감소·가독성 향상, (3) Spring/Android 공식 지원으로 실무 적용이 수월함.

### 6-2. 도입을 제안할 때

- **리스크 완화**: "한 번에 안 바꿔도 됩니다. 새 API·새 모듈만 코틀린으로 하고, 기존 자바는 그대로 둬도 됩니다."
- **이득**: "null 때문에 터지는 버그를 컴파일 단계에서 줄일 수 있고, DTO·유틸 코드가 짧아져서 리뷰가 빨라집니다."
- **학습 비용**: "자바 하시는 분이면 문법 차이만 익히면 되고, 우리가 Step 1~8 정리해 둔 걸로 온보딩할 수 있습니다."

### 6-3. 실무에서 정리

| 상황 | 어떻게 쓰는지 |
|------|----------------|
| 면접·채용 | "JVM 호환, null 안전, 간결함, Spring 지원" 네 가지를 예시와 함께 말할 수 있게 정리 |
| 기술 선택 회의 | 점진적 전환 가능·버그 감소·가독성·공식 지원을 근거로 제안 |
| 레거시 전환 | 새 코드만 코틀린, 기존 자바와 상호호출 가능함을 강조 |

---

## 7. 주의점

- "자바 대체"가 아니라 **자바와 함께** 쓸 수 있는 언어라고 두는 게 현실적이다. 팀·회사에선 **점진적 도입**을 전제로 말하는 경우가 많다.
- 학습 비용이 있다. **왜 쓰는가**를 공유해 두면, "문법만 외우는 게 아니라 이렇게 쓸모가 있다"는 맥락을 줄 수 있다.
